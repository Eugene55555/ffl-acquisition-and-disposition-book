# Почему я сделал этот блог статическим

Большинство по привычке берут WordPress. Он работает, но тянет за собой базу
данных, рантайм PHP, обновления плагинов и вечные security-патчи. Для
контентного сайта это избыточно.

## Что даёт статика

- **Скорость** — страницы это чистый HTML, отдаваемый с CDN. Загрузка меньше секунды.
- **Безопасность** — взламывать нечего. Нет формы входа, нет SQL, нет рантайма.
- **Дёшево** — GitHub Pages бесплатен для публичных репозиториев.
- **Переносимость** — весь сайт это git-репозиторий. Перенеси куда угодно.

## Как работает публикация

Пост — это Markdown-файл в `content/posts/`. Когда он попадает в `main`,
GitHub Action собирает сайт и публикует его. Никаких кликов по дашборду —
источник правды это репозиторий.

В этом и смысл: блог это просто текст и git.
